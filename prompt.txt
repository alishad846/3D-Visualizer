Backend — six endpoints needed for this page:
If any already exist in analyticsController.js with correct SQL derived from schema.sql, do not rewrite them. Only implement what is missing or incorrect. For each endpoint, read schema.sql to confirm the columns exist before writing the query.
The six endpoints are:

GET /api/analytics/project/:projectId/overview — five stat card metrics: total scans all-time, unique visitors all-time, avg session duration, AR rate for last 28 days. Ownership verified via JOIN to projects table. Returns: { totalScans, uniqueVisitors, avgSessionSeconds, arRate }.
GET /api/analytics/project/:projectId/realtime — count of distinct ip_hash values where scanned_at is within last 30 minutes, across all products in the project. Returns only { activeNow } — no per-minute sparkline, no country breakdown. Those are product-level details not needed here.
GET /api/analytics/project/:projectId/trend?range=7d|28d|90d — daily scan counts grouped by date for current period and previous period of equal length. Current period: last N days. Previous period: the N days immediately before that. Returns { current: [{day, scans, uniqueVisitors, avgSession, arRate}], previous: [{day, scans}] }. Default range if missing or invalid: 28d.
GET /api/analytics/project/:projectId/geo?range=7d|28d|90d — scan counts grouped by country_code across all products in project, filtered by range, ordered by scans descending, limit 50. Returns { countries: [{countryCode, scans, percentage}] }. Percentage is each country's share of total. Use window function for percentage calculation.
GET /api/analytics/project/:projectId/sources?range=7d|28d|90d — scan counts grouped by referrer_type for current period and previous period. Calculate trend percentage in controller. Returns { sources: [{referrerType, scans, percentage, trend}] }.
GET /api/analytics/project/:projectId/products?range=7d|28d|90d&sort=scans|unique_visitors|avg_session|ar_rate|last_scanned — one row per product in the project with aggregated stats. Include a second query for 7-day sparkline data. Merge sparkline into product rows in the controller. Mark products as dead if they have zero scans in the last 30 days. Use a whitelist map for the sort column — never interpolate user input directly into SQL. Returns { products: [{id, name, thumbnailUrl, slug, totalScans, uniqueVisitors, avgSessionSeconds, arRate, lastScanned, isDead, sparkline}] }.

For every endpoint: wrap in try/catch, log errors with [Analytics] prefix, return 500 with { error: 'Analytics query failed' } on failure. Never expose raw SQL errors or stack traces to the client.
All six routes must go through the existing auth middleware. Add them to server/src/routes/analytics.js following the exact same pattern already used in that file.
